# Proposed Title and Chinese Abstract

## English Title

**Does the Judge Prefer English? Evaluating Code-Switching Invariance in LLM-as-a-Judge**

## Chinese Abstract

随着大语言模型越来越多地被用作开放式生成任务的自动评测器，LLM-as-a-Judge 的可靠性已成为模型评测中的关键问题。已有研究表明，LLM 裁判可能受到答案位置、长度、表达风格和提示模板的影响，但其在多语言和中英混写场景下是否保持一致判断仍缺乏轻量、可复现的检验。本文提出一个面向 LLM 裁判的代码混合不变性评测方案 Judge-CS。我们基于 LLMBar 中具有客观偏好标签的指令跟随样本，构造英文、中文和中英混写三种语义保持变体，并对每个样本进行答案位置交换和翻译等价偏好探测。理想的裁判模型应在语言变体之间维持相同的优劣判断，并在语义等价的英文与中文答案之间给出平局而非系统性偏向某一语言。本文使用多个可通过 API 调用的模型作为裁判，报告准确率、位置一致性、语言不变性翻转率和英文偏好率等指标。该方案无需训练模型，也不依赖本地大规模算力，适合作为低成本、可扩展的 LLM 评测可靠性研究。初步实验显示，即使在较小样本上，不同模型在中文和中英混写条件下也可能出现判断翻转或格式不稳定现象，说明自动评测结果可能混入语言呈现方式带来的偏差。本文旨在为多语言环境下 LLM-as-a-Judge 的使用提供一个简单可行的诊断工具，并为后续构建更稳健的自动评测协议提供依据。
